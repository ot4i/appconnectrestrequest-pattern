Parameters for the App Connect REST Request Pattern pattern

Basic Pattern Parameters
Pattern Parameter Default Description
App Connect

The Application drop-down lists the available SaaS applications. Select an application and depending on the value chosen, the Action drop-down shows the available actions and the object they act upon.
Create IIB Flow Selected

Use the checkbox to define whether or not you would like the pattern to generate an IIB message flow. The pattern parameters beneath this checkbox relate to the IIB message flow, so they are only enabled when this checkbox is selected.
IIB Input Protocol HTTP

This drop-down enumerates the available protocols (choice of HTTP, MQ or File) used to drive the IBM Integration Bus message flow.
HTTP URL Path Suffix /Example

This parameter is enabled if HTTP was selected for the IIB Input Protocol. If the URL that you want the IIB message flow to listen on is http://hostname[:port]/[path], then this property will specify the /path
MQ Input Queue EXAMPLE.IN

This parameter is enabled if MQ was selected for the IIB Input Protocol. The IIB message flow will take its input messages from this MQ Input Queue.
MQ Output Queue EXAMPLE.OUT

This parameter is enabled if MQ was selected for the IIB Input Protocol. The IIB message flow will write its output messages to this MQ Output Queue.
File Input Directory C:\ExampleInput

This parameter is enabled if File was selected for the IIB Input Protocol. The IIB message flow will read its input files from this Input Directory.
File Output Directory C:\ExampleOutput

This parameter is enabled if File was selected for the IIB Input Protocol. The IIB message flow will write its output files to this Output Directory.
File Output Filename output.txt

This parameter is enabled if File was selected for the IIB Input Protocol. The IIB message flow will write its output files with this Output Filename.
IIB Transform Type Mapping node

This drop-down lets you choose the type of IIB transformation node which will be inserted into the message flow and used to demonstrate assigning values. The choices are Compute node, JavaCompute node, Mapping node, or no transformation node at all.

All the pattern parameters are listed with their default values. Where no default is indicated, the default value is an empty string.
